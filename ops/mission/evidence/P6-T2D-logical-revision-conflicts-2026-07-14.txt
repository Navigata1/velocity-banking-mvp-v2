Implemented contract
- A new migration starts existing install snapshots at revision epoch 0 and requires
  each distinct operation to advance its owner/snapshot stream by exactly one.
- Exact immutable operation retries return their original structured receipt before
  stale or gap checks, so a later accepted revision cannot be rolled back.
- Distinct operations serialize on owner plus snapshot identity. Stale revisions use
  SQLSTATE IS001 and gapped revisions use IS002 before profile, snapshot, or audit mutation.
- The revision-free six-argument RPC is removed in the same coordinated migration.
- Native outbox v2 persists per-stream allocated and acknowledged revisions even when
  empty, keeps conflicts at the FIFO head, and blocks new work until recovery.
- Browser sync uses an owner/snapshot IndexedDB read-write transaction for cross-tab
  allocation and persists immutable pending work before the RPC.
- Browser retries drain older pending work and do not report success until the caller's
  current normalized snapshot intent is itself acknowledged.
- Legacy native v1 queue state fails closed rather than guessing a server revision.

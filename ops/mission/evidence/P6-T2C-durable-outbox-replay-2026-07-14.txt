P6-T2C - Durable owner-scoped Expo snapshot outbox and foreground replay
Date: 2026-07-14 America/Chicago
Branch: codex/mission/p6-native-outbox-replay

Implemented
- Sync presses prepare an immutable owner-bound request and persist it before checking connectivity.
- Owner UUIDs resolve to one canonical queue key; embedded owner mismatches and malformed queues fail closed.
- The bounded FIFO retains one operation identity and payload across reloads and ambiguous RPC failures.
- Replay stops at the first failure, removes only the acknowledged head, and shares one in-flight flush per owner.
- Duplicate operation IDs are rejected before persistence.
- Sign-in, network reconnection, and app activation trigger Expo Go-compatible foreground replay.
- Account changes cancel late replay notices and the server-side expected-owner check prevents cross-account mutation.
- Background success/failure is surfaced to Settings through uniquely identified, consumable owner-scoped notices.
- SecureStore stale-generation cleanup is best-effort after manifest commit, preventing false queue success/failure messages.
- Offline copy says only "Saved on this device" because Expo web uses localStorage while native uses SecureStore.

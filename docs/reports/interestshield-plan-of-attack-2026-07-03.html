Backend Plan

Recommended First Lane: Supabase

Best fit for serious user-owned app state: Auth, Postgres, RLS, migrations, row ownership, simulation history, learning progress, and account deletion.

  • Create dedicated Supabase project for InterestShield.
  • Apply reviewed owner-scoped schema in a controlled environment.
  • Wire Auth only after RLS policies and migration tests are reviewed.
  • Add user-owned snapshots, simulation runs, learning progress, export, import, and deletion.

Secondary Lane: Cloudflare

Keep Cloudflare for edge/API value after ownership rules are stable, not as a shortcut around data security.

  • Use Workers for API boundaries if needed.
  • Use D1 only through server-side owner verification.
  • Use Durable Objects only where coordination or session-state semantics justify them.
  • Do not let browser clients write directly to D1.
